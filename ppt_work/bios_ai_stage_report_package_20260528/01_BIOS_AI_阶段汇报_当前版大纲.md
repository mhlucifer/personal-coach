## Slide 12：下一步计划

**页面结论**：下一步聚焦 SMBIOS 可观测问题，沉淀开发生成和测试修复两条 Agent 流程。

开发阶段工作流：
- 需求 / 客户规则 → 规格文档 → opencode skills → AI 辅助生成代码 → 自动化验证 → 人工验收 → 上库。
- 适用：规则清晰、字段边界明确、输出可比对的 SMBIOS 模块。

测试问题工作流：
- 自动验证发现差异 → Agent 读取日志和代码 → 定位上报路径 → 输出修复建议 → 工程师确认 → 复测回归。
- 适用：字段缺失、显示异常、客户可见信息不一致、配置开关导致上报缺失。

边界：
- 不做泛化 BIOS Bug 自动修复；启动失败、硬件链路异常、时序不稳定、偶发平台问题不纳入本轮闭环。

验收：
- 选取 2-3 类 SMBIOS 显示 / 字段问题，保留验证报告、定位记录、修复建议、人工确认和复测结果。
