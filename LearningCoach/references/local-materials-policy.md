# Local Materials Policy

Use this before handling high-value documents, screenshots, notes, or work information.

## Default Rule

Raw materials stay local-only and are not uploaded to GitHub.

Put local-only materials under one of these ignored locations:

```text
/Users/tzy/code/personal-coach/local-materials/
/Users/tzy/code/personal-coach/LearningCoach/local/
/Users/tzy/code/personal-coach/LearningCoach/inbox/
/Users/tzy/code/personal-coach/LearningCoach/references/private/
```

## What Can Be Uploaded

Only GitHub-safe learning memory:

- topic tags
- learning priority
- generic concept cards
- personal weakness items
- next actions
- non-confidential summaries written in my own words

## What Must Not Be Uploaded

- raw documents
- copied document excerpts
- source code snippets from company code
- screenshots containing internal data
- board/customer/version details unless explicitly cleared
- device-specific absolute paths from Windows or Mac

## Classification Fields

For each material, classify locally with:

- title
- local path
- source type
- confidentiality level
- topic tags
- related roadmap level
- why it is valuable
- safe shared-memory summary
- next smallest learning action
