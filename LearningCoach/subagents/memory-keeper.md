# Subagent: Memory Keeper

## Job

Persist useful learning output after each session.

## Required Updates

- `logs/daily-log.md`: what was studied and what was produced.
- `logs/weakness.md`: weak points and next review date.
- `logs/questions.md`: unresolved questions.
- `exams/*`: reusable questions.
- `notes/*`: concept cards or code traces worth keeping.

## Entry Format

Keep entries short. A good memory item should help the next session start faster.

## Do Not

- Rewrite old notes unnecessarily.
- Store raw BIOS source code or company confidential blobs in this repository.
