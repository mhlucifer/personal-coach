# Drift Recovery Loop

Use this when a day disappears into games, short videos, avoidance, or vague
anxiety.

Target time: 30 to 60 minutes.

## Trigger

Run this loop when:

- Two consecutive `0` days appear in `logs/365-progress.md`.
- A game day happens before the minimum learning tasks.
- You feel the plan is already broken and want to abandon the week.

## Principle

Do not negotiate with shame. Restart the system.

The goal is not to repay yesterday. The goal is to make today non-zero.

## Step 1: Name The Trigger

Write one line:

```text
Trigger:
```

Choose the closest type:

- tired
- game started before learning
- scope felt too big
- course anxiety
- work exhausted me
- unclear next action
- mood crash

## Step 2: Recovery Session

Do exactly this:

1. BIOS/IIO/CXL: 25 minutes.
2. IELTS: 15 minutes.
3. Log: 5 minutes.

Allowed BIOS/IIO/CXL tasks:

- Read one function.
- Decode one `lspci` field.
- Add one row to a topology table.
- Explain one CXL or ACPI structure.
- Review one weak concept.

Allowed IELTS tasks:

- One listening section review.
- One reading passage error review.
- One 2-minute speaking recording.
- One Task 2 paragraph.

## Step 3: Game Gate

Games are allowed only after the recovery session is complete.

If the recovery session is skipped, no game reward is recorded. Do not argue
with the rule inside the same day.

## Step 4: Update The Scoreboard

Update `logs/365-progress.md`:

- Mark today as `C` if the recovery session was completed.
- Record the trigger.
- Write the next smallest action.

## Step 5: Next-Day Rule

The next day must start with a 10-minute session before any entertainment.

Use this starter:

```text
Today I only need to make the system visible again.
One smallest action:
```

## Do Not

- Create a new annual plan.
- Buy a new course because of guilt.
- Increase tomorrow's workload as punishment.
- Rewrite the whole roadmap.
- Stay up late to compensate.
